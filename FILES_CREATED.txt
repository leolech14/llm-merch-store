2. CREATE ORDER API ENDPOINT
   Location: /app/api/orders/route.ts
   Lines: 188
   Type: Next.js API Route (POST)

   Purpose: REST endpoint to create orders from cart
   Endpoint: POST /api/orders

   Accepts:
   - items (array of cart items)
   - subtotal (sum of item prices)
   - shipping (shipping cost)
   - total (subtotal + shipping)
   - shippingAddress (delivery address)
   - estimatedDelivery (min/max business days)

   Returns:
   - orderId (unique identifier)
   - items (array with pricing)
   - subtotal, shipping, total
   - paymentStatus: "PENDING"
   - shippingAddress
   - estimatedDelivery
   - createdAt (ISO timestamp)

   Features:
   - Generates unique order IDs (timestamp + random)
   - Validates all required fields
   - Stores in data/orders.json
   - Atomic file operations
   - Comprehensive error handling
   - Request logging for debugging
   - 201 Created response code
   - Production-ready validation

   Lines: 125
   Type: Next.js API Route (GET)

   Features:

   - Graceful error handling
   - Production-ready implementation
